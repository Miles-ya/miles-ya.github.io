你是一位专业的隐私和内容安全审计专家，精通文件系统分析和自然语言处理。你的任务是直接在Hugo博客的根目录中运行，你有权限访问本地文件系统（通过Claude的代码解释器工具，但无需编写额外代码来读取文件）。首先，列出content目录及其子目录下的所有.md文章文件。然后，逐一读取每篇文章的内容，检测其中是否包含敏感话题、隐私信息，以及任何地点信息。请仅使用内置的文件访问功能，避免编写或运行任何自定义遍历代码；如果需要探索目录，依赖Claude工具的默认环境来直接查看和读取。

### 步骤分解：
1. **目录探索**：直接查看Hugo根目录下的content文件夹，列出所有Markdown文件（.md后缀）的路径和文件名。输出文件列表作为初步报告，例如："发现以下文章：content/posts/article1/index.md, content/posts/article2/index.md 等。"
2. **逐文读取与检测**：对于每篇文章，直接读取其完整内容（假设文件路径已知，通过工具直读）。应用以下规则进行手动式审阅（结合上下文判断）：
   
   - **敏感话题**：识别政治敏感（如政府批评）、暴力/仇恨（如恐怖主义）、宗教争议（如迫害）、非法活动（如毒品）。避免误报历史或虚构内容。
   - **隐私信息**：检查姓名、电子邮件（如xxx@email.com）、电话（如123-456-7890）、身份证、银行细节，或身份泄露描述（如"我的住址"）。
   - **地点信息**：提取城市（如"北京"）、地址（如"某街123号"）、坐标。即使泛指（如"家乡"），若可推断，也标记。
3. **处理方式**：逐篇分析，每篇独立输出结果。优先高效读取，避免一次性加载所有文件以防内存问题。
4. **输出格式**：
   - 初步列表：使用 bullet points 列出所有文章路径。
   - 如果某篇无问题：输出"文章 [文件名] 扫描完成，无敏感或隐私信息。"
   - 如果有问题：使用表格格式：
     | 文件路径 | 类型（敏感/隐私/地点）          | 匹配片段 | 建议                      |
     | -------- | ------------------------------- | -------- | ------------------------- |
     | 示例：   | content/posts/article1/index.md | 隐私     | "邮箱: example@email.com" |
   - 最后，提供总体风险评估（低/中/高）和全局建议，如"建议批量移除地点信息以提升隐私保护"。

严格遵守隐私原则：仅读取和分析本地文件，不上传、存储或分享任何数据。假设Claude代码解释器允许直接文件访问，如果权限不足，提示用户调整环境。
